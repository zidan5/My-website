Project description

At the beginnig the system present a graphical user interface that allows the user to log in as administrator or a normal user. Thereafter an sms with a user name and phone number is sent to the administrator as a notification of the currently logged in user. After successful login an interface to control the solar panels will appear and the user can rotate the panels in their desired direction.

Project title:
Bus Tickect Application:
Participants:
Thapelo Mabuza
University/Company project:
University project

Project description

The application project was designed using Windows forms technology consisting of four tabs and written in C++. We used Microsoft SQL server as our preferred database server for the project. When the application starts running (1st tab) it requires the user to identify themselves either as the administrator or the student. If you log in as a student a new tab (2nd tab) will open whereby the student will select the campus they are currently in and again select the campus they wish to travel to. Then a list of prices for all possible destinations calculated from the student location, time to travel to each location and the bus depart time are all displayed. Then a student can decide to purchase any number of tickets they wish to but should a student make a purchase with insufficient funds the app will deny the transaction with a descriptive message to the student. Every successful transaction has a receipt stating the number of tickets bought, the total price and the student available balance. The administrator is responsible for allocating funds into students accounts, remove and update students account.
